HyperAgent Live
Hyperface · AI Labs · 2024–Present
The Problem

Fintechs and banks want to build AI agents — for customer conversations, decision workflows, lead qualification, collections, and more — but engineering these from scratch is expensive and slow. Every team ends up reinventing the same plumbing: LLM orchestration, tool integrations, compliance guardrails, knowledge bases, and deployment infrastructure.

What I Built

A DIY AI agent builder purpose-built for fintech — partners configure and deploy agents for any use case, embedding them across any surface they own: a chatbot in their app, a decision agent in their backend, an IVR, a PWA, or a CRM workflow. Each agent is given a role, tools (API calls or MCP endpoints), a knowledge base, guardrails, and a prompt — all without writing code.

Where It Is

Voice AI — one of the first use cases built on HyperAgent — is live with a fintech client in Hindi, with issuers in active discussion across multiple use cases. The platform is built to support any agent type a fintech partner wants to configure and ship.

Campaign Co-Pilot Building
Hyperface · AI Labs · 2025–Present
The Problem

Banks run campaigns on fixed calendars with multi-team analysis cycles. Strategies don't adapt between campaigns; new banks often lack historical data to start well; and there's no feedback loop that makes each campaign smarter than the last.

The Architecture

A five-layer system: a common fintech knowledge base, issuer-specific context, a campaign learning layer, a behavioural profile (channel effectiveness, timing patterns, segmentation — auto-updated with 90-day decay), and a live data layer. The key insight: generate an intelligent strategy without relying on historical bank data, by leveraging fintech ecosystem patterns and lifecycle stage behaviour. A chat co-pilot lets campaign heads question, give feedback, and refine in real time.

The Bet

Every campaign feeds a compounding intelligence layer — making the next strategy incrementally better. The goal is a system that gets genuinely smarter over time, not just faster.

PaceIQ Live
Personal project · React + Vite · FastAPI · Claude Vision · 2026
The Problem

Built for runners who track everything but understand nothing from their data. You finish a run, your watch shows 47 metrics, and you still don't know if the session actually helped. Most training apps tell you what happened; none tell you what it means for someone like you.

What It Does

After every run, you upload your splits screenshot and paste your training plan. PaceIQ analyses actual vs planned pace, heart rate zones, and interval execution — then gives you a personalised breakdown: what went well, what to fix, how to recover, and what to focus on next. It knows your profile — age, weight, city, altitude, shoe mileage, target races — so every insight is specific to you, not generic advice. Over time it tracks aerobic base development and keeps you honest about whether you're actually ready for race day.

Stack

React + Vite frontend, FastAPI backend, SQLite, Claude Vision API for splits analysis.

Calorie Tracker Live
Personal project · Claude + Notion API · 2026
The Problem

I'm training for a half marathon while trying to hit a weight goal. Every calorie app I tried died within a week — barcode scanning, manual logging, fixed calorie targets — none of it fit how I actually eat or train.

What I Built

A conversational nutrition tracker: describe or photograph a meal, and Claude identifies the food, looks up accurate nutrition data, and logs it directly to a Notion database — zero manual entry. Targets aren't flat; they adapt to the day type: rest days at 1,900 kcal, gym days at 2,100, easy runs at 2,200, long runs at 2,300. Macro splits shift too — more carbs on run days, more protein on gym days. A daily summary tracks calories and protein per meal slot against the day's goal.

Why It Works for Me

No app to open, no barcodes, no friction. Just describe the meal. The system handles the rest — and it's been running daily since I started training.

Restaurant Reel Analyser Building
Personal project · Discord · Claude Vision · Whisper · Notion API · 2026
The Problem

I find most restaurants through Instagram reels — but the gap between discovery and actually going is where everything falls apart. I'd save the reel, forget about it, or manually copy the name into my notes app and lose context. The log existed but it was always incomplete, always out of date, always friction.

What I Built

A Discord bot that turns any Instagram reel link into a structured Notion entry — automatically. Drop a link in a channel, and the bot downloads the reel, extracts key frames via ffmpeg, transcribes the audio with Whisper, and sends everything to Claude Vision to pull out the restaurant name, area, cuisine, dish, and veg/non-veg status. The result gets saved to a Notion database with zero manual input. If extraction fails, the bot asks you to fill in what's missing — so nothing slips through.

Stack

Python · Discord.py · yt-dlp · ffmpeg · Claude Vision API · OpenAI Whisper · Notion API
